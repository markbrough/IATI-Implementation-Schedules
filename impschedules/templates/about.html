Process

Analysing and processing the common standard schedules has required a detailed look at each donor's schedule, as well as interpreting them when donors have filled them out in different ways or understood the various options differently.

The original implementation schedules

The implementation schedules for the common standard were published in individual Excel files, containing three main sheets: general, agency, and activity-level information.

  • General

    Approach to publication: includes timeliness and frequency, license, initial publication dates, scope of publication.

  • Agency

    Agency-level publication: includes country budgets, organisation documents.

  • Activity

    Activity-level publication: includes information about when the organisation will (or won't) be compliant with each field in the standard.

Templates

While the common standard has a template for implementation schedules, several different templates exist, and donors further added to this complexity by modifying the templates, changing options, and adding and deleting rows – in all, we counted eleven different versions. In some cases, importing the schedules proved difficult because of this. In addition, some donors saved their templates in PDF format which made it impossible to automatically parse. In these cases, it was necessary to copy the same data into new spreadsheets. While the data was thoroughly checked, it is possible that some human error will have resulted. Some interpretation was necessary to ensure consistency and comparability across them (you can read more about the assumptions made above).

Importing

The schedules were automatically parsed and imported into this application. Publish What You Fund staff then checked the resulting data to ensure that it had been parsed correctly and that it made sense.

  • Select a file

    The user can select a schedule from any publicly-accessible URL. This could be the OECD/DAC common standard website, the IATI website, or the donor's own website.

    Where the original schedules could not be automatically parsed (e.g. because they were in PDF), a new file was created and uploaded to the Publish What You Fund website.

  • Successfully parsed

    The schedule was successfully parsed and is presented to the user to check and correct.

  • Check fields

    Make sure that all information has been parsed correctly, ensure that compliance status is consistent with notes, and score for partially compliant fields if the publisher had been overly conservative.
